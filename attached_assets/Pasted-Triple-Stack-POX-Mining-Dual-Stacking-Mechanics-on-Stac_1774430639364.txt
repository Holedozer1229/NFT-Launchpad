Triple Stack POX Mining: Dual Stacking Mechanics on Stacks (March 2026)

This guide outlines how to build a triple‑stack yield machine that combines:

1. Bitcoin L1 → mint sBTC (1:1 Bitcoin‑backed asset)
2. Stacks → participate in Dual Stacking to earn boosted BTC rewards
3. Solana → bridge sBTC via Wormhole NTT and feed into OIYE Quantum Sentinel for automated compounding

The result is a self‑reinforcing loop: Bitcoin mining economics → Dual Stacking yields → cross‑chain private transfer → autonomous yield optimization.

---

1. Dual Stacking on Stacks (as of March 2026)

Dual Stacking is a non‑consensus extension that allows:

· sBTC holders to earn base BTC rewards (paid in sBTC, redeemable 1:1 for BTC).
· STX stackers to pair their locked STX with sBTC and multiply their BTC rewards (up to 10×).

Cycle structure – 2 weeks (aligned with PoX cycles).
Reward formula (simplified):

w_i = \frac{B_i \cdot (1 + M \cdot \sqrt{r_i})}{n}

where:

· B_i = average sBTC balance of user i during the cycle
· r_i = min(1, stacked STX / sBTC) – ratio capped at 1
· M = boost multiplier (default 9 → up to 10×)
· n = normalisation factor across all participants

All rewards are paid in sBTC at the end of each cycle.

---

2. Triple‑Stack Architecture

The flow:

1. BTC → sBTC (Peg‑in)
   Send native BTC to a Stacks‑generated peg‑in address; sBTC is minted on Stacks.
2. Enroll in Dual Stacking
   Deposit sBTC and optionally lock STX to apply a boost.
3. Earn boosted BTC rewards
   After a 2‑week cycle, claim sBTC rewards.
4. Bridge sBTC to Solana
   Use Wormhole Native Token Transfer (NTT) to move sBTC to Solana.
5. Feed into OIYE Factory
   Deposit into the OIYE Quantum Sentinel for automated yield farming.

The system can be automated with a bot that monitors cycle boundaries, claims rewards, and bridges them.

4.1 Minting sBTC (Peg‑in)

The peg‑in is initiated from Bitcoin L1. The peg‑in address is generated by the Stacks signer set. You can obtain it via a function call to the sBTC contract:

```javascript
import { callReadOnlyFunction, cvToValue, standardPrincipalCV, uintCV } from '@stacks/transactions';
import { StacksTestnet } from '@stacks/network';

const network = new StacksTestnet(); // or mainnet
const sbtcContract = 'SP2PABAF9FTAJYNFZH93XENAJ8FVY99RRM50D2JG9.sbtc-token';

// Get a fresh peg‑in address for the user (simplified)
const getPegInAddress = async (userAddress) => {
  const result = await callReadOnlyFunction({
    contractAddress: sbtcContract.split('.')[0],
    contractName: sbtcContract.split('.')[1],
    functionName: 'get-peg-in-address',
    functionArgs: [standardPrincipalCV(userAddress)],
    senderAddress: userAddress,
    network
  });
  return cvToValue(result);
};
```

Then send BTC to that address using your Bitcoin wallet. Once the Bitcoin transaction is confirmed (usually 6 confirmations), the sBTC will be minted to your Stacks address. You can monitor the status via the sBTC contract.

4.2 Enrolling in Dual Stacking

After sBTC is minted, enrol for the current cycle. The enrolment function typically takes the amount of sBTC and optionally a lock of STX.

```javascript
import { makeContractCall, broadcastTransaction, uintCV, someCV, noneCV } from '@stacks/transactions';

const dualStackingContract = 'SP2PABAF9FTAJYNFZH93XENAJ8FVY99RRM50D2JG9.dual-stacking-v1';

// sBTC amount in micro‑units (1 sBTC = 1e8 micro‑sBTC)
const sbtcAmount = 100_000_000n; // 1 sBTC

// Optional STX amount in micro‑STX (1 STX = 1e6 micro‑STX)
const stxAmount = 10_000_000n; // 10 STX

const txOptions = {
  contractAddress: dualStackingContract.split('.')[0],
  contractName: dualStackingContract.split('.')[1],
  functionName: 'enroll',
  functionArgs: [
    uintCV(sbtcAmount),
    stxAmount > 0 ? someCV(uintCV(stxAmount)) : noneCV()
  ],
  senderKey: privateKey, // your private key
  network
};

const tx = await makeContractCall(txOptions);
const result = await broadcastTransaction(tx, network);
console.log('Enrolment transaction:', result.txid);
```

4.3 Claiming Rewards at Cycle End

At the end of each cycle, call claim to receive your sBTC rewards.

```javascript
const claimRewards = async () => {
  const txOptions = {
    contractAddress: dualStackingContract.split('.')[0],
    contractName: dualStackingContract.split('.')[1],
    functionName: 'claim',
    functionArgs: [],
    senderKey: privateKey,
    network
  };
  const tx = await makeContractCall(txOptions);
  return broadcastTransaction(tx, network);
};
```

You can detect cycle boundaries by monitoring the PoX cycle number (increases every ~2100 Bitcoin blocks). Alternatively, the contract may provide a read‑only function get-current-cycle.

After claiming rewards, you’ll have sBTC on Stacks. Use Wormhole’s NTT to transfer to Solana.

First, approve the bridge to spend your sBTC:

```javascript
const approveTx = await makeContractCall({
```

```javascript
const nttTransfer = await makeContractCall({
  contractAddress: bridgeContractAddress.split('.')[0],
  contractName: bridgeContractAddress.split('.')[1],
  functionName: 'transfer',
  functionArgs: [
    uintCV(amountToBridge),

```

On the Solana side, you’ll need to run a relayer or use an existing service to claim the tokens. The Wormhole SDK provides functions to redeem the transfer on Solana.

4.5 Feeding into OIYE Quantum Sentinel

Once sBTC is on Solana, you can deposit it into the OIYE Factory to participate in the Quantum PvP Arena for optimal APY allocation.

```typescript
import { Connection, PublicKey, Transaction } from '@solana/web3.js';
import { getAssociatedTokenAddress, createAssociatedTokenAccountInstruction, createTransferInstruction } from '@solana/spl-token';

const OIYE_FACTORY = new PublicKey('OIYEfAcTorYxxxxxxxxxxxxxxxxxxxxxxxxxxxxx');

async function depositToOIYE(sbtcMint: PublicKey, amount: number, userWallet: Keypair) {
  const connection = new Connection('https://api.mainnet-beta.solana.com');
  const userAta = await getAssociatedTokenAddress(sbtcMint, userWallet.publicKey);
  const factoryAta = await getAssociatedTokenAddress(sbtcMint, OIYE_FACTORY, true);

  const instructions = [];
  const userAtaInfo = await connection.getAccountInfo(userAta);
  if (!userAtaInfo) {
    instructions.push(createAssociatedTokenAccountInstruction(userWallet.publicKey, userAta, userWallet.publicKey, sbtcMint));
  }

  instructions.push(createTransferInstruction(userAta, factoryAta, userWallet.publicKey, amount));

  const tx = new Transaction().add(...instructions);
  const signature = await connection.sendTransaction(tx, [userWallet]);
  await connection.confirmTransaction(signature);
}
```

After deposit, the OIYE Sentinel will automatically route funds to the highest‑yield strategies.

5. Automation Bot (Node.js)

A bot can run continuously, handling the entire triple‑stack pipeline:

```javascript
import { StacksNetwork } from '@stacks/network';
import { DualStackingManager } from './dual-stacking.js';
import { BridgeManager } from './bridge.js';
import { OIYEManager } from './oiye.js';

class TripleStackBot {
  constructor(config) {
    this.config = config;
    this.network = new StacksNetwork(config.stacksNetwork);
    this.dual = new DualStackingManager(config.dualStacking);
    this.bridge = new BridgeManager(config.bridge);
    this.oiye = new OIYEManager(config.oiye);
    this.lastCycle = 0;
  }

  async start() {
    setInterval(() => this.run(), this.config.interval * 1000);
    await this.run(); // immediate run
  }

  async run() {
    try {
      // 1. Check current cycle
      const currentCycle = await this.dual.getCurrentCycle();
      if (currentCycle === this.lastCycle) return; // still same cycle

      // 2. If a new cycle started, claim previous cycle's rewards
      if (this.lastCycle !== 0) {
        const claimed = await this.dual.claimRewards();
        if (claimed) {
          // 3. Bridge claimed sBTC to Solana
          const amount = await this.dual.getClaimableBalance();
          if (amount > 0) {
            const bridgeTx = await this.bridge.transferToSolana(amount);
            // 4. Wait for bridge completion, then deposit to OIYE
            const solanaTx = await this.bridge.waitForRedeem(bridgeTx.id);
            await this.oiye.deposit(solanaTx.receivedAmount);
          }
        }
      }

      // 5. Ensure we are enrolled for the new cycle
      const enrolled = await this.dual.isEnrolled();
      if (!enrolled) {
        const sbtcBalance = await this.dual.getSbtcBalance();
        if (sbtcBalance > 0) {
          await this.dual.enroll(sbtcBalance, this.config.stxStackingAmount);
        }
      }

      this.lastCycle = currentCycle;
    } catch (err) {
      console.error('Bot error:', err);
    }
  }
}

// Configuration
const bot = new TripleStackBot({
  interval: 3600, // check every hour
  stacksNetwork: 'mainnet',
  dualStacking: {
    privateKey: 'your_stacks_private_key',
    sbtcContract: 'SP2PABAF9FTAJYNFZH93XENAJ8FVY99RRM50D2JG9.sbtc-token',
    dualContract: 'SP2PABAF9FTAJYNFZH93XENAJ8FVY99RRM50D2JG9.dual-stacking-v1',
    stxStackingAmount: 10_000_000n // 10 STX
  },
  bridge: {
    wormholeBridgeStacks: 'SP...wormhole-bridge',
    wormholeBridgeSolana: 'worm2...',
    solanaRpc: 'https://api.mainnet-beta.solana.com'
  },
  oiye: {
    factoryAddress: 'OIYEfAcTorYxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',
    sbtcMint: 'sBtcMintOnSolana...'
  }
});

bot.start();
```

6. Security & Assumptions

· Smart contracts – Assumed to be audited and deployed by the Stacks ecosystem.
· Signer set – The peg‑in/peg‑out is managed by a decentralised set of signers; trust is placed in that set (as per Stacks PoX).
· Bridge – Wormhole NTT relies on validators; we assume the bridge is secure.
· Private keys – Must be stored securely; the bot uses a single Stacks key for all operations. Consider using a multi‑sig or hardware wallet for high‑value operations.
· Fees – All operations incur Stacks (STX) fees and Solana transaction fees; ensure the bot wallet has sufficient STX for transactions.
